Hardware Constraint: 8GB RAM optimization required

EXECUTIVE SUMMARY
CINEMATIQ is an industry-grade, AI-powered recommendation platform that provides personalized movie, web series, and TV show recommendations without requiring user authentication. Built on the MovieLens dataset and optimized for limited hardware, it employs hybrid recommendation algorithms (collaborative filtering + content-based filtering) to deliver Netflix-quality recommendations in a lightweight, accessible format.

MACHINE LEARNING MODEL ARCHITECTURE
Dataset Processing (MovieLens 239MB)
Data Structure:

movies.csv: movieId, title, genres
ratings.csv: userId, movieId, rating, timestamp
tags.csv: userId, movieId, tag, timestamp
links.csv: movieId, imdbId, tmdbId

Preprocessing Steps:

Data Cleaning:

Remove duplicates
Handle missing values (impute or drop)
Parse genres (split by pipe)
Extract year from title
Normalize ratings (0-5 scale)


Feature Engineering:

Genre one-hot encoding
TF-IDF for title text
User preference vectors
Item popularity scores
Temporal features (release year, rating year)
Average rating per movie
Rating count per movie


Data Splitting:

Train: 80%
Validation: 10%
Test: 10%
Stratified by rating distribution



Recommendation Algorithm Design
Hybrid Approach (Optimized for 8GB RAM):
Component 1: Collaborative Filtering (60% weight)

Algorithm: Matrix Factorization (Truncated SVD)
Dimensions: 50 latent factors
Memory footprint: ~200MB
Implementation: Scikit-learn TruncatedSVD
Benefits: Captures user-item patterns
Handles sparse matrix efficiently

Component 2: Content-Based Filtering (40% weight)

Algorithm: Cosine Similarity on TF-IDF
Features: Genre + Title keywords
Memory footprint: ~150MB
Implementation: Scikit-learn TfidfVectorizer + cosine_similarity
Benefits: Works for new items
Genre-aware recommendations

Model Optimization Techniques:

Dimensionality Reduction:

Use TruncatedSVD instead of full SVD
Limit latent factors to 50
Sparse matrix operations
CSR format for storage


Efficient Storage:

Precompute similarity matrices
Store as sparse matrices
Cache top-N recommendations
Use pickle for serialization


Inference Optimization:

Batch recommendation generation
Precompute user/item vectors
Use NumPy vectorization
Approximate nearest neighbors (Annoy library)



Model Training Pipeline
Step 1: User-Item Matrix Construction

Create sparse rating matrix (users × movies)
Apply min-max normalization
Convert to CSR format

Step 2: Collaborative Model Training
Algorithm: TruncatedSVD
n_components: 50
n_iter: 20
random_state: 42
Training time: ~2-3 minutes
Step 3: Content Feature Extraction
TF-IDF Parameters:
max_features: 1000
ngram_range: (1, 2)
stop_words: 'english'
min_df: 2
Step 4: Similarity Computation

Compute item-item cosine similarity
Keep top 50 similar items per item
Store in sparse matrix

Step 5: Model Persistence

Save models: pickle format
File size: <500MB total
Load time: <5 seconds

Recommendation Generation Logic
For New Users (Cold Start):

Show trending content (popularity-based)
Allow genre selection
Use content-based filtering on selected genres
Gradually build user profile

For Returning Users (Session-Based):

Track viewed items in localStorage
Build temporary user vector
Apply hybrid recommendation
Blend: 60% collaborative + 40% content

Recommendation Scoring:
Final_Score = (0.6 × Collaborative_Score) + (0.4 × Content_Score)
Apply diversity penalty to avoid filter bubble
Boost recent/trending items by 10%
Filter out already viewed items
Return top 20 recommendations
